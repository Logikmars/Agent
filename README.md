# pc-folder-agent

## Обновление: память и быстрый поиск

- Долгосрочная память хранится локально в `agent-index.sqlite` в таблицах `memories` и `memory_search_terms`.
- Агент автоматически подмешивает релевантные воспоминания в контекст вопроса и сохраняет краткое событие диалога после ответа.
- Для явного управления памятью добавлены tools `recall_memory` и `remember_memory`.
- Поиск файлов ускорен через локальный инвертированный индекс `file_search_terms`: имена, пути, расширения и текст чанков токенизируются при сканировании.
- Старый `LIKE`-поиск оставлен как fallback, если по терм-индексу ничего не найдено.

Локальный read-only AI-агент на Node.js/TypeScript. Он индексирует одну разрешенную папку `WATCH_DIR`, хранит индекс в SQLite, строит дерево файлов, извлекает текст из поддерживаемых документов и отвечает на вопросы через локальный API или CLI. Изображения анализируются только по конкретному запросу через OpenAI Vision.

## Возможности

- рекурсивное сканирование `WATCH_DIR`;
- SQLite-индекс файлов, папок и текстовых чанков;
- watcher на `chokidar` для добавления, изменения и удаления файлов;
- API на Fastify: `/health`, `/files`, `/tree`, `/rescan`, `/ask`, `/analyze-image`;
- CLI-команды: `ask`, `rescan`, `tree`;
- OpenAI Responses API с function/tool calling;
- строгая read-only модель: агент не удаляет, не переименовывает и не изменяет файлы.

## Установка

```bash
npm install
```

## Настройка

Создайте `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Пример:

```env
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-5.5
WATCH_DIR=./workspace
MAX_FILE_SIZE_MB=25
MAX_IMAGE_SIZE_MB=20
PORT=3000
DATABASE_PATH=./agent-index.sqlite
```

`OPENAI_API_KEY` можно временно оставить пустым: сервер запустится, а `/ask` вернет локальный обзор без вызова модели.

## Запуск

```bash
npm run dev
```

При старте приложение создаст `WATCH_DIR`, просканирует папку, запустит watcher и API на `http://127.0.0.1:3000`.

Админка доступна здесь:

```text
http://127.0.0.1:3000/admin
```

В ней можно смотреть индекс файлов, дерево папки, запускать rescan, задавать вопросы агенту и отправлять выбранное изображение на анализ.

Для production-сборки:

```bash
npm run build
npm start
```

## Поддерживаемые форматы

Текст извлекается из:

`.txt`, `.md`, `.json`, `.csv`, `.log`, `.html`, `.css`, `.js`, `.ts`, `.jsx`, `.tsx`, `.py`, `.java`, `.php`, `.xml`, `.yaml`, `.yml`, `.pdf`, `.docx`.

Изображения:

`.png`, `.jpg`, `.jpeg`, `.webp`, `.gif`.

В MVP текст из изображений локально не извлекается. Картинка отправляется в OpenAI только если пользователь спрашивает о конкретном изображении.

## Безопасность

- агент работает только внутри `WATCH_DIR`;
- каждый путь проходит через `path.resolve` и проверку, что он не выходит за `WATCH_DIR`;
- операции записи, удаления, перемещения и переименования пользовательских файлов не реализованы;
- в модель не отправляются все файлы подряд: агент выбирает релевантные файлы через tools;
- большие файлы пропускаются по лимитам `MAX_FILE_SIZE_MB` и `MAX_IMAGE_SIZE_MB`;
- `.env`, `.git`, `node_modules`, `dist`, `build`, `.next`, `.cache` игнорируются.

## Ограничения MVP

- нет UI/Electron;
- нет авторизации пользователей;
- нет облачной синхронизации;
- нет OCR для изображений без OpenAI Vision;
- поиск по чанкам основан на SQLite `LIKE`, без embeddings.
